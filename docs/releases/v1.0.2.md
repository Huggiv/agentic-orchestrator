# Release v1.0.2

Release date: 2026-06-26

## Highlights

- Chat now responds with concise backend LLM guidance even when no Jira ticket is provided.
- Operators can cancel chat-associated jobs directly from chat messages.
- Jira bulk ticket table usability is improved with internal scrolling and sticky headers.
- Model selection reliability is improved with fallback loading and manual refresh actions.

## Included Changes

### Backend

- Added no-ticket LLM response generation for both `/api/chat/message` and `/api/chat/message/stream`.
- Added chat cancel endpoint: `POST /api/chat/cancel/{job_id}`.
- Added cancellation guard to allow chat cancellation only for chat-triggered jobs.
- Expanded backend tests for no-ticket LLM behavior and chat cancel guard behavior.

### Frontend

- Added inline chat job controls with per-job Cancel actions and status transitions.
- Added manual model refresh buttons in:
  - Trigger Automation form
  - Bulk Trigger modal
  - Floating chat composer
- Improved model loading with direct API fallback and shared error/loading state.
- Added internal Jira table scrolling and sticky headers for bulk selection workflows.

## API Updates

- New endpoint: `POST /api/chat/cancel/{job_id}`
- Updated chat behavior:
  - No-ticket prompts now produce concise LLM-generated assistance.

## Notes

- This release focuses on chat workflow usability and operational controls, with emphasis on reliability during model selection and multi-ticket orchestration.
